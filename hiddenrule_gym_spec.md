# HiddenRule-Gym 环境规格 v0.1

**定位**:纯Python文本POMDP合成环境,为PS-GRPO提供 (a) latent state真值 (b) 可参数化的部分可观测度与随机性 (c) 程序化判分的可验证特征 (d) belief与credit归因的ground truth。
**设计核心决策**:所有任务族的隐藏变量 θ 取自**有限可枚举假设空间**(|Θ| 控制在 10³–10⁶),使得精确贝叶斯后验 b*_t(θ) = P(θ | h_t) 可以由模拟器端动态规划/枚举精确计算。这一个决策同时解锁:真值belief、探针题自动生成与判分、belief/action credit真值标注、观测互信息的精确测量、贝叶斯最优策略oracle。理论部分的所有量都因此可测。

---

## 1. 统一形式化

每个实例由种子 σ 生成:
- 隐藏变量 θ ∈ Θ(规则/地图/偏好),episode内不变(非平稳变体除外);
- 状态 s_t = (θ, z_t),z_t 为可见动态部分(位置、回合数、已获物品等);
- 观测 o_t = Noise(Leak(s_t, a_{t-1}; ω); ε_o),文本形式;
- 转移 z_{t+1} ~ T(z_t, a_t; ε_t);
- 终端奖励 R ∈ {0, 1}(或分级 {0, 0.5, 1}),中途无任务奖励(稀疏设定是实验前提)。

三个全局旋钮(所有任务族同一语义):

| 旋钮 | 含义 | 范围 | 理论对应量 |
|---|---|---|---|
| ω(泄露度) | 单步观测携带的关于θ的信息量档位 | {0,1,2,3}档,每族定义见§2 | 控制 I(o_t; θ \| h_{t-1}),模拟器精确计算并随日志输出 |
| ε_o(观测噪声) | 观测内容以概率ε_o被翻转/腐蚀/置空 | [0, 0.4] | 定理中观测通道噪声项 |
| ε_t(转移噪声) | 动作以概率ε_t产生非预期转移 | [0, 0.3] | 定理中环境随机性项 |

附加旋钮:d(干扰率)——每步以概率d注入与θ无关的干扰文本段(模拟记忆压力),信息论上零信息但消耗上下文。区分"信息稀缺"与"上下文拥挤"两种失效来源,消融要用。

## 2. 三个任务族

### 2.1 RuleInduction(静态假设推断,"科学家"型)
- **θ**:一个DSL生成的谓词 f: 属性元组 → {accept, reject}。属性空间 = 颜色×形状×数量×材质(各4–6值)。DSL文法:字面量比较、合取(≤3项)、析取(≤2项)、异或、计数条件。枚举后 |Θ| ≈ 10³–10⁵(按文法深度分难度档)。
- **交互**:agent每回合可 (a) test(元组) 获得反馈,(b) 中途做 predict(元组) 自测(不计任务分),(c) 终局 answer:对16个hold-out元组给出分类,≥14对记成功。
- **ω档位**:3=反馈给出"哪个属性违反了规则"的定位提示;2=accept/reject;1=延迟批反馈(每3次test才统一返回结果,打乱对应关系);0=只返回"本批accept数量"。
- **horizon**:20–40 turns。
- **考察点**:主动探索(问什么组合最有信息量)+ belief压缩(记忆里维护候选规则集的摘要)。

### 2.2 TrapGrid(动态状态追踪,"探险家"型)
- **θ**:N×N网格(N=7–11)的隐藏布局:陷阱位置、钥匙-门配对、传送门映射、开关-桥联动。布局从带约束的程序生成器采样,保证可解;枚举编码后 |Θ| ≈ 10⁴–10⁶(用因子化后验:各机关独立/弱耦合,后验按因子精确维护)。
- **交互**:move/inspect/use;踩陷阱回起点(不终止),步数预算内到达终点记成功。
- **ω档位**:3=视野半径2且标注格子类型;2=视野半径1;1=只报告"当前格+撞墙/触发事件"的结果反馈;0=事件反馈也模糊化("发生了某事")。
- **ε_t语义**:移动滑移(向垂直方向偏移);开关触发失败。
- **horizon**:40–100 turns(N和机关数控制)。
- **考察点**:空间belief的长程维护——早期inspect到的信息在几十步后仍需可用,直接压测记忆策略;且这是三族中唯一"动作影响信息获取路径"的族,探索-利用耦合最强。

### 2.3 PrefOracle(多session偏好推断,"管家"型)
- **θ**:模拟用户在K个属性维度上的偏好权重向量(每维取离散档 {-2..+2}),|Θ| = 5^K,K=6–8。
- **交互**:分3–20个session,每session若干轮;agent可闲聊提问(用户以softmax(β·效用)风格给出含噪偏好信号)、可推荐;**session间上下文强制清空,只允许记忆摘要 m 跨session携带**——这是三族中对记忆策略最"外科手术式"的隔离设计;终局推荐top-3命中真实top-1记成功。
- **ω档位**:3=用户直接陈述某维偏好;2=对具体item给评价(多维耦合信号);1=只有比较偏好("A比B好");0=只有接受/拒绝推荐。
- **ε_o语义**:用户理性温度β反比于ε_o(β→∞为完全理性);另有p_contradict概率给出自相矛盾的表述。
- **非平稳变体**(P3阶段选做):每session偏好向量以小概率漂移一维,测试记忆的更新/遗忘。
- **horizon**:总计30–100 turns。
- **考察点**:跨session记忆credit(直接对标Memory-R2的问题设定)+ 含噪证据聚合。

**三族覆盖矩阵**:静态θ+主动查询(2.1)/ 动态z+空间记忆(2.2)/ 跨session+含噪社会信号(2.3);理论定理在三族上的预测各不相同(见§5),这是"理论有内容"的证据。

## 3. 可验证特征接口(预测充分性奖励的判分入口)

```python
env.verifiable_features(k: int) -> list[Feature]
# Feature = {qid, question_text, answer_space, answer}
# 语义:关于未来k步内可由模拟器裁决的事实,答案在step时由env内部记录
```

每族的特征类型(全部是二元/多选,规则程序判分,零LLM参与):
- RuleInduction:"若下一步test(x),反馈将是accept吗?"、"元组y在规则下的真实标签?"(仅当后验已收敛到可判定时列入);
- TrapGrid:"格子(i,j)是陷阱吗?"、"若向北移动,会触发事件吗?"、"钥匙k对应的门在东半区吗?";
- PrefOracle:"用户对item z的评价将≥中性吗?"、"维度c的偏好为正吗?"。

**关键设计**:特征在每个turn由env基于当前真实s_t自动生成一批(8–16个),按"当前精确后验下的可判定性"过滤——后验下答案概率∈[0.9,1]∪[0,0.1]的才算"可判定",避免让agent为本质随机的问题背锅。判分给出逐特征0/1,预测奖励 r_pred(t) = 可判定特征上的加权准确率增量。此过滤规则本身进论文方法节。

## 4. Belief探针与credit真值

- **探针题库**(仅评测,绝不进训练信号):对θ的MAP查询、边缘概率查询("陷阱在(3,4)的概率>50%吗")、蕴含查询。答案由精确后验 b*_t 计算,同时给出agent应有的"理性不确定"标签——探针不仅测准确率,还测**校准**(agent的口头概率 vs 精确后验概率,算ECE)。每instance自动生成30–50题,按turn索引,评测时对齐时间点。
- **credit真值标注**:失败episode逐turn计算两个量:(a) belief缺口 = KL(b*_t ‖ 探针反演出的agent belief);(b) 动作缺口 = 精确后验下贝叶斯最优动作价值 − agent动作价值(小实例上用后验加权DP算Q*)。失败归因 = 两缺口首次超阈值者。这给PS-GRPO的belief/action credit解耦提供了可量化的归因准确率指标——并发工作没有任何一家能报这个数。

## 5. 理论挂钩(定理要用到的可测量)

| 定理中的量 | 环境如何提供 |
|---|---|
| 预测损失 ε | §3特征上的经验预测损失 |
| 单步信息泄露 | I(o_t; θ \| h_{t-1}),由精确后验的熵差直接算 |
| 环境随机性项 | ε_t, ε_o 为已知真值,代入上界画理论曲线 vs 实测曲线 |
| 充分性基线 | 精确后验b*_t作为"完美记忆"上界;截断历史作为下界 |

预注册的理论预测(写进论文,增强可信度):(1) 固定ε_o,方法增益随ω下降而先增后减(信息太少时预测目标也学不动);(2) ε_t增大时,k步预测horizon的最优值应缩短;(3) PrefOracle的跨session设定下,预测奖励对比结果奖励的样本效率优势最大。

## 6. 实例生成、切分与规模

- 每族按(|Θ|档 × ω × ε_o × ε_t)网格生成;规则/布局/偏好在train与test间**假设空间划分不相交**(按DSL子树/生成器种子段隔离),防记忆化;
- 训练集每族3000实例,验证500,测试500;失效边界实验用旋钮网格上每格100测试实例;
- 观测文本模板配3套改写变体,防止对模板过拟合;
- 全环境无外部依赖(标准库+numpy),单实例step延迟<5ms,支持进程池并发数千实例;env状态可序列化(支持Memory-R2式局部re-rollout:从任意turn的快照分叉)。

## 7. 接口约定(与统一env层对齐)

```python
class HiddenRuleEnv:
    def reset(self, seed, config) -> Observation
    def step(self, action: str) -> (Observation, done, info)
    def verifiable_features(self, k) -> list[Feature]      # 训练奖励用
    def probe_bank(self) -> list[Probe]                     # 仅评测
    def oracle(self) -> OracleHandle                        # 仅评测/标注:精确后验、Q*、MI
    def snapshot(self) / restore(state)                     # re-rollout用
```

动作以自然语言+结构化标记混合(`<action>test(red, cube, 2)</action>`),解析失败返回可读错误并计一次无效步(无效步率也是被试指标)。
